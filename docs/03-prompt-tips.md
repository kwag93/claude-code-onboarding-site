# 03. 프롬프트 팁

Claude Code와 효과적으로 대화하는 방법입니다.

## 핵심 원칙

### 1. 맥락을 충분히 제공하기

**Before** ❌

```
로그인 기능 만들어줘
```

**After** ⭕

```
JWT 기반 로그인 기능을 만들어줘.
- 기존에 User 모델이 있고 email, password_hash 필드가 있어
- FastAPI 쓰고 있고, /api/v1 prefix 사용 중
- 토큰 만료는 24시간으로
```

### 2. 원하는 결과물을 명확히

**Before** ❌

```
이 코드 좀 봐줘
```

**After** ⭕

```
이 코드를 리뷰해줘. 특히:
- 성능 문제가 있을 수 있는 부분
- 에러 처리가 부족한 곳
- 네이밍 개선이 필요한 부분
```

### 3. 점진적으로 대화하기

한 번에 모든 것을 요청하기보다, 대화를 통해 발전시키세요.

```
> API 엔드포인트 설계해줘

[Claude가 설계안 제시]

> 좋은데, 페이지네이션 추가해줘

[Claude가 수정]

> 에러 응답 형식도 통일하고 싶어

[최종 완성]
```

## 상황별 프롬프트 템플릿

### 새 기능 개발

```
[기능명]을 개발하려고 해.

배경:
- [왜 이 기능이 필요한지]
- [관련된 기존 코드나 모듈]

요구사항:
- [구체적인 요구사항 1]
- [구체적인 요구사항 2]

제약사항:
- [기술적 제약이나 규칙]
```

### 버그 수정

```
[버그 현상] 문제가 있어.

재현 방법:
1. [단계 1]
2. [단계 2]

예상 동작: [원래 어떻게 되어야 하는지]
실제 동작: [실제로 어떻게 되는지]

관련 코드: [파일명이나 함수명]
```

### 코드 설명 요청

```
[파일명/함수명]의 동작을 설명해줘.

특히 이해가 안 되는 부분:
- [구체적인 질문 1]
- [구체적인 질문 2]

[주니어 개발자도 이해할 수 있게 / 간단히 요약해서] 설명해줘
```

### 리팩토링 요청

```
[파일/함수]를 리팩토링하고 싶어.

현재 문제:
- [문제점 1: 예를 들어 "함수가 너무 길다"]
- [문제점 2: 예를 들어 "중복 코드가 많다"]

목표:
- [원하는 상태]

주의사항:
- 기존 테스트가 통과해야 함
- [다른 주의사항]
```

## 효과적인 대화 패턴

### 패턴 1: "먼저 계획을 세워줘"

복잡한 작업은 바로 코드 작성을 요청하지 말고, 계획부터 요청하세요.

```
> 결제 시스템을 리팩토링하려고 해.
> 바로 코드 수정하지 말고, 먼저 어떤 순서로 진행하면 좋을지 계획을 세워줘.

[Claude가 계획 제시]

> 좋아, 2번 단계부터 시작하자
```

### 패턴 2: "왜 그렇게 했는지 설명해줘"

코드를 받기만 하지 말고, 이유를 물어보세요.

```
> 왜 여기서 useCallback을 썼어?
> 이 패턴의 장단점이 뭐야?
> 다른 방법은 없었어?
```

### 패턴 3: "더 나은 방법이 있어?"

제안받은 코드도 개선의 여지가 있을 수 있습니다.

```
> 동작은 하는데, 이게 최선이야?
> 성능이나 가독성 면에서 더 개선할 수 있는 부분 있어?
```

### 패턴 4: "제약 조건 추가하기"

결과물이 마음에 안 들면 제약을 추가해보세요.

```
> 더 간단하게 할 수 없어?
> 외부 라이브러리 없이 해줘
> 한 함수가 20줄을 넘지 않게 해줘
```

## 피해야 할 패턴

### ❌ 너무 모호한 요청

```
이 코드 좀 고쳐줘  (뭘 고치라는 건지?)
```

### ❌ 너무 많은 것을 한 번에

```
로그인, 회원가입, 비밀번호 찾기, 소셜 로그인 전부 만들어줘
(하나씩 요청하세요)
```

### ❌ 맥락 없는 코드 조각

```
왜 안 돼?
(에러 메시지, 파일명, 무엇을 하려 했는지 설명해주세요)
```

### ❌ 결과만 사용하고 이해하지 않기

```
[코드 받음] → [바로 복붙] → [나중에 버그 발생]
(왜 이렇게 작성했는지 이해하고 사용하세요)
```

## 한국어 vs 영어

둘 다 잘 이해하지만:

- **한국어**: 편하게 대화하기 좋음, 뉘앙스 전달 쉬움
- **영어**: 기술 용어가 자연스러움, 코드 주석/커밋 메시지 작성 시 유리

섞어 써도 됩니다:

```
이 함수를 pure function으로 리팩토링해줘.
side effect 없이, immutable하게.
```

## 자주 쓰는 유용한 표현들

| 상황               | 표현                        |
| ------------------ | --------------------------- |
| 간단하게 원할 때   | "가장 심플한 방법으로 해줘" |
| 설명이 필요할 때   | "주석 달아서 설명해줘"      |
| 안전하게 원할 때   | "기존 동작은 유지하면서"    |
| 확인하고 싶을 때   | "이렇게 이해한 게 맞아?"    |
| 선택지가 필요할 때 | "몇 가지 방법을 제안해줘"   |
| 단계별로 원할 때   | "하나씩 천천히 진행하자"    |
| 범위를 좁힐 때     | "지금은 이것만 집중해줘"    |

## 마무리

좋은 프롬프트의 핵심은 **명확함**과 **구체성**입니다.

"Claude가 내 마음을 읽어주겠지"보다는
"Claude가 이 정보만으로 정확히 이해할 수 있을까?"라고 생각해보세요.

연습이 필요합니다. 처음엔 어색해도 점점 자연스러워질 거예요.
