# 06. 에이전트 팀과 코워크 (2026년 2월 업데이트)

Claude Code와 Claude Desktop의 최신 협업 기능을 실제 업무 관점에서 정리한 문서입니다.

---

## 이번 버전에서 달라진 점

2026년 1월~2월 사이에 협업 기능이 빠르게 확장되었습니다.

| 날짜 | 변경 내용 | 실무 영향 |
| --- | --- | --- |
| 2026-01-12 | 코워크 연구 미리보기가 Max 플랜(Claude Desktop, macOS)에서 시작 | 긴 작업을 백그라운드로 돌리는 방식 도입 |
| 2026-01-16 | 코워크가 Pro 플랜으로 확대 | 개인 사용자의 접근성 증가 |
| 2026-01-23 | 코워크가 Team/Enterprise로 확대 | 팀 단위 운영 가능 |
| 2026년 2월 초 | 오푸스 4.6 공개 + 에이전트 팀 공개(실험 기능) | 병렬 작업 품질과 작업 안정성 개선 |

---

## 에이전트 팀이란?

에이전트 팀은 Claude Code에서 **여러 에이전트를 하나의 팀으로 묶어 동시에 작업**시키는 기능입니다.
문서 기준으로는 아직 **실험 기능**입니다.

### 언제 쓰면 좋은가

- 한 작업을 역할별로 병렬 분해해야 할 때
- 코드 수정 + 테스트 + 문서화처럼 결과물이 여러 종류일 때
- 대화 한 번으로 큰 작업을 끝내고 싶을 때

### 활성화 방법

`~/.claude/settings.json` 또는 프로젝트별 `.claude/settings.json`에 아래를 추가합니다.

```json
{
  "env": {
    "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
  }
}
```

이후 Claude Code에서 `/agents` 명령으로 팀을 생성하고, 팀에 전달할 공통 목표를 지정합니다.

### 실사용 예시

```text
목표: 결제 모듈 리팩토링
팀 구성:
- 분석 담당: 현재 결제 흐름과 리스크 식별
- 구현 담당: 결제 로직 분리 및 인터페이스 정리
- 검증 담당: 회귀 테스트 케이스 추가

출력 형식:
1) 변경 요약
2) 파일별 수정 목록
3) 테스트 결과
4) 남은 위험 항목
```

## 코워크란?

코워크는 Claude Desktop에서 **대화와 별도로 장시간 작업을 백그라운드로 수행**하도록 만든 기능입니다.
코딩 외 일반 지식 업무(분석, 정리, 자료 작성)까지 확장된 형태입니다.

### 현재 지원 범위 (2026년 2월 기준)

- 요금제: Max, Pro, Team, Enterprise
- 환경: Claude Desktop의 macOS 환경에서 시작
- 동작 특성: 작업 진행 중에도 다른 일을 할 수 있고, 완료 시 결과를 다시 확인

### 시작 순서

1. Claude Desktop에서 모드 선택 메뉴로 이동
2. `코워크` 모드 선택
3. 큰 작업을 한 번에 넘기되, 출력 형식까지 명시
4. 완료 후 결과 검수 및 후속 수정 지시

### 코워크 요청 템플릿

```text
목표:
- 이번 주 매출/비용/이슈를 종합해 주간 운영 보고서 작성

입력 자료:
- sales.xlsx
- support_tickets.csv
- release_notes.md

반드시 포함:
1) 핵심 지표 5개
2) 전주 대비 변화 원인
3) 다음 주 액션 아이템 3개
4) 경영진용 5문장 요약
```

---

## 오푸스 4.6과 코워크

오푸스 4.6은 에이전트형 코딩 작업 안정성, 장문 추론, 고난도 산출 품질 개선이 강조되었습니다.
또한 파워포인트 애드인에서는 소넷 4.5와 오푸스 4.6 전환이 가능합니다.

주의:

- 공식 문서에 "오푸스 4.6이 코워크 전용 기능을 직접 추가했다"는 문구가 명시된 것은 아닙니다.
- 다만 모델 성능 향상과 애드인 모델 선택 확장을 고려하면, 코워크 결과물 품질 향상에 유리하다고 해석할 수 있습니다.

운영 팁:

- 탐색/초안은 소넷 계열로 시작하고
- 최종 검토/난도 높은 판단은 오푸스 4.6으로 마감
- 결과물 제출 전 반드시 사람이 숫자/출처/정책 준수 여부를 검수

## 엑셀에서 Claude: 실전 가이드

### 1) 설치

- 개인 사용자: 오피스 추가 기능 스토어에서 Claude 추가 기능 설치
- 조직 사용자: 마이크로소프트 365 관리 센터에서 관리자가 조직 배포

### 2) 작업 방식

워크북 안에서 Claude 패널을 열고 범위를 지정해 요청합니다.

#### 추천 요청 패턴

```text
이 시트의 B열~F열을 기준으로
1) 누락값과 이상치를 먼저 찾고
2) 월별/카테고리별 집계를 만든 뒤
3) 경영진이 읽을 7줄 요약을 작성해줘.
수식은 왜 그렇게 만들었는지 한 줄 설명도 포함해줘.
```

```text
이 데이터로 피벗 테이블 2개를 제안해줘.
- 하나는 매출 추세
- 하나는 지역별 수익성
각 피벗에서 봐야 할 핵심 해석 포인트를 3개씩 적어줘.
```

### 3) 결과 검수 체크리스트

- [ ] 통화/날짜 형식이 원본 규칙과 같은가?
- [ ] 분모가 0인 계산이 안전하게 처리됐는가?
- [ ] 피벗/차트 범위에 누락 행이 없는가?
- [ ] 요약 문장이 실제 수치와 일치하는가?

### 4) 버전 참고

엑셀 도움말 문서의 모델 표기는 시점에 따라 빠르게 바뀝니다.
문서와 실제 화면의 모델 선택 항목이 다르면 **화면 기준으로 우선 확인**하세요.

## 파워포인트에서 Claude: 실전 가이드

### 1) 설치

- 개인 사용자: 오피스 추가 기능 스토어에서 설치
- 조직 사용자: 관리자 일괄 배포 권장

### 2) 슬라이드 작성 흐름

1. 발표 목적/청중/시간(예: 10분)을 먼저 전달
2. 슬라이드 목차 초안 생성
3. 슬라이드별 핵심 메시지 1개씩 고정
4. 마지막에 톤/길이/중복만 정리

#### 추천 요청 패턴

```text
대상: 임원진
목적: 2026년 1분기 운영 성과 보고
분량: 10장 이내

요청:
1) 장표 구조(제목 + 핵심 메시지) 제안
2) 각 장표의 말할 거리 3개
3) 마지막 장표에 의사결정 요청 항목 2개 추가
```

```text
현재 8번 장표가 텍스트가 많아.
핵심 문장 3개만 남기고,
나머지는 발표자 노트로 내려줘.
톤은 단정하고 수치 중심으로 유지해줘.
```

### 3) 모델 운영 팁

- 빠른 수정 반복: 소넷 계열
- 메시지 정제/고난도 서술: 오푸스 4.6
- 최종본은 발표자가 직접 읽어보며 문장 길이와 용어 통일 점검
